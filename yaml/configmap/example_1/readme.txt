1)

В данном примере выполняя команду "kubectl apply -f configmap.yaml"

Мы создаем объект типа ConfigMap с названием "my-test-configmap" в который запишется ключ "size" со значением "512MB"


2)

Далее выполняя команду "kubectl apply -f deployment.yaml"

Мы создаем deployment, который создает под с контейнером с Nginx.

Внутри окружения контейнера при его старте будет создана переменная "size", значение для которой будет взято из конфигмапа с именем "my-test-configmap" из ключа "size"



То есть путь такой:

ConfigMap "my-test-configmap", значение ключа "size" -------> переменная окружения "size" внутри контейнера
